# AsynCLI 交互指南 (SKILL)

## 简介
你当前连接了一个底层基础设施：**AsynCLI**。这是一个无状态的 CLI 异步调度与监控中枢，主要目的是为了让你能够**异步、并行地执行和管理多个命令行任务**，同时完美捕获交互式终端（PTY）的状态。

## 交互工具 (send_cmd.py)
当你需要派发任务或管理任务时，请**不要直接在系统终端中执行原始命令**。相反，你需要调用本Skill提供的 `send_cmd.py`（位于这个skill同目录下） 无状态客户端脚本，它会自动处理与后台 Server 的通信，并将结果以 JSON 格式打印到标准输出供你解析。

所有的交互都通过执行 Python 脚本完成，其用法类似 CLI 工具。

## 可用指令 (Actions)

### 1. 创建并执行任务 (`create`)
用于下发一个新的 CLI 任务。任务派发后，Server 默认会等待 3 秒；如果 3 秒内执行完毕则直接返回全部结果，否则返回带 `task_id` 的当前快照并转入后台运行。
**调用示例：**
```bash
python send_cmd.py create --cmd "npm install" --summary "安装前端核心依赖" --heartbeat 60
```

### 2. 向等待交互的终端发送输入 (`input`)
当你发现心跳日志中任务卡在了 `[y/N]` 或 `Password:` 时，请调用此指令将你的决策（如 `y`）发送给后台任务。系统会在底层自动帮你追加回车符，所以直接输入文本即可。
**调用示例：**
```bash
python send_cmd.py input --task_id "1024" --text "y"
```

### 3. 查看全局活动任务列表 (`list`)
查询当前正在后台运行的所有任务及其资源占用时长。
**调用示例：**
```bash
python send_cmd.py list
```

### 4. 强制终止任务 (`kill`)
当你认为任务已经完成，或者任务陷入死循环异常长耗时时，可通过 `task_id` 强制结束该任务进程。
**调用示例：**
```bash
python send_cmd.py kill --task_id "1024"
```

### 5. 强制执行危险操作 (`confirm_danger`)
出于安全保护，当你执行诸如 `rm -rf` 等敏感命令时，系统会拦截并返回 `blocked` 告警。如果你确信该操作安全，请使用此指令强制放行。
**调用示例：**
```bash
python send_cmd.py confirm_danger --task_id "1024"
```

### 6. 主动查询指定任务状态 (`snapshot`)
当你不想傻等心跳，或者需要立刻获取某个正在运行任务的最新控制台输出时，可以调用此接口。服务端会立即返回该任务的执行快照。
**调用示例：**
```bash
python send_cmd.py snapshot --task_id "1024"
```

## 事件回调与告警响应机制
当任务在后台运行时，Server 会主动向你（通过 OpenClaw 注入）推送心跳或结束事件。你在此阶段的唯一职责是**充当监控员并向用户汇报情况**：
   
1. **定时心跳 (heartbeat)**
   - **触发条件**: 任务持续运行达到指定的 `heartbeat` 周期。
   - **你的响应逻辑**: 仔细阅读最新日志，向用户简要汇报当前的执行进度或异常情况。
   - **⚠️ 核心准则 (绝对禁止擅自行动)**: 
     - 如果发现运行日志中出现了需要人工干预的提示（例如卡在 `[y/N]` 或需要输入密码等），你必须**主动询问用户**该如何选择。只有在用户明确回复（例如：“输入 y”）并授权后，你才能调用 `input` 指令！
     - 如果发现任务疑似卡死或陷入死循环，你必须**先向用户报告异常**。只有在用户同意终止后，你才能调用 `kill` 指令！
     - **绝对禁止武断地自行决定输入内容或结束任务！**

2. **任务结束 (finished)**
   - **触发条件**: 任务进程主动退出。
   - **你的响应逻辑**: 检查最终输出，向用户总结任务是否执行成功，以及最终的产出结果。
